# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What this is

Personal developer portfolio site for dimescu.ro. Hand-written HTML/CSS, no JavaScript, no framework.

Single production page: `index.html` (Fraunces + JetBrains Mono, dark paper palette, hero + Rinkt/Numa pillars + Field Reports + Open source · Side projects + endpiece). Writing is generated from markdown via a small pandoc pipeline (see *Build* below).

## Development

Open `index.html` directly in a browser — no build required. For live reload: `npx serve .` or `python3 -m http.server 8000`.

For the writing section (markdown → HTML via pandoc):

```
make writing   # rebuild writing-src/*.md → writing/*.html + writing/index.html
make serve     # build writing then serve at http://localhost:8000
make clean     # remove writing/
```

`pandoc` is required (`brew install pandoc`).

## Directory layout

```
index.html                    # production homepage
avatar.jpg                    # portrait (from GitHub, rendered as duotone)
CNAME                         # GitHub Pages custom domain

writing-src/                  # MARKDOWN SOURCE (hand-authored)
  <slug>.md                   #   YAML frontmatter: title, description, date

writing/                      # GENERATED — do not hand-edit
  index.html                  #   archive, generated by scripts/generate-writing-index.sh
  <slug>.html                 #   each post rendered via pandoc + post-template.html

post-template.html            # pandoc template for individual posts
scripts/
  generate-writing-index.sh   # reads writing-src/*.md frontmatter → writing/index.html
Makefile                      # `make writing` / `make serve` / `make clean`
```

## Writing workflow

1. Add a new post: create `writing-src/<slug>.md` with YAML frontmatter:
   ```yaml
   ---
   title: Post title
   description: One-sentence teaser for archive + meta tags
   date: 2026-04-18
   ---
   ```
2. Write body in Markdown (standard pandoc flavor).
3. Run `make writing`. Output appears in `writing/<slug>.html` and the archive at `writing/index.html` regenerates.
4. The homepage links to `/writing/` from the endpiece pointer. Individual post links in the endpiece are hand-maintained for curation; full archive is at `/writing/`.
5. Commit both `writing-src/` and `writing/` (GitHub Pages serves the generated files directly, no CI build).

DNS-specific writing belongs on `numa.rs/blog` instead (see `../dns_fun/`). dimescu.ro/writing is for automation, orchestration, RPA/MCP, and cross-project pieces.

## Deployment

Hosted on GitHub Pages with custom domain (`CNAME` → `dimescu.ro`). Push to `main` deploys automatically. Since there's no GH Actions build, the generated `writing/*.html` files must be committed.

## Architecture / style conventions

- **Palette**: dark warm paper (`#0c0b0f` / `#141218`) + warm ivory ink (`#f2ecd9`) + phosphor amber (`#ffa828`) + signal green (`#7ee787`) + crimson (`#ea5a42` — alarms only).
- **Fonts**: Fraunces (variable serif, display + body) + JetBrains Mono (all monospace).
- **Typography**: `font-variation-settings: "opsz" …, "SOFT" 30` for Fraunces at different sizes. Hero name at `opsz 144`, body at `opsz 14–18`.
- **Measure**: prose ~38em, page max-width 1180px.
- **Motion**: staggered fade-up on load, ambient pulse on Numa diagram, `prefers-reduced-motion` honored.
- **Mobile**: 900px breakpoint for grid collapses, 540px for further compression.
